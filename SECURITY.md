# Политика безопасности

## Поддерживаемые версии
Мы выпускаем исправления безопасности только для последней стабильной версии бота. Если вы используете более старую версию, рекомендуется обновиться до актуальной.

| Версия | Поддерживается |
|--------|----------------|
| 1.2.x  | ✅ Да          |
| < 1.2  | ❌ Нет         |

## Сообщение об уязвимости
Если вы обнаружили уязвимость в безопасности, пожалуйста, сообщите нам как можно скорее. Мы серьёзно относимся к безопасности и постараемся оперативно отреагировать.

### Как сообщить
1. **Не создавайте публичный Issue** для уязвимостей.
2. Отправьте описание проблемы одним из способов:
   - Создайте **конфиденциальный Issue** на GitHub с пометкой `[SECURITY]` в заголовке (мы постараемся скрыть его от публичного просмотра, но GitHub не гарантирует полную приватность).
   - Напишите на электронную почту: Cluelesstech@yandex.ru.
3. В сообщении укажите:
   - Краткое описание уязвимости.
   - Шаги для воспроизведения (если возможно).
   - Потенциальное влияние на систему.
   - Ваши контактные данные для обратной связи (по желанию).

### Процесс
- Мы подтвердим получение вашего сообщения в течение **48 часов**.
- Проведём анализ и определим критичность проблемы.
- Свяжемся с вами для уточнения деталей, если потребуется.
- После исправления мы выпустим обновление и уведомим вас.
- По согласованию с вами, мы опубликуем информацию об уязвимости после выхода патча.

## Рекомендации
- Всегда используйте последнюю версию бота.
- Храните файл `.env` с токенами в надёжном месте и никогда не публикуйте его.
- Регулярно обновляйте зависимости (`npm update`), чтобы получать исправления безопасности.

## Благодарности
Мы ценим ответственное раскрытие уязвимостей и благодарим всех, кто помогает делать проект безопаснее.
